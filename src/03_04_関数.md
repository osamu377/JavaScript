## I. 用語の整理

<ruby>関数<rt>かんすう</rt></ruby>について説明する前に、整理しておきたい言葉があります。前に出てきた言葉ですが、ここで復習しておきましょう。

プログラミングを勉強していると、漢字が並んでいて<ruby>難<rt>むずか</rt></ruby>しく感じますよね。**「<ruby>演算<rt>えんざん</rt></ruby>子」** と **「式」** と **「文」** です。これらは、プログラミングの **「文法（ぶんぽう）」** の名前です。そして、どれもプログラムの<ruby>動<rt>うご</rt></ruby>きを書くためのものです。

日本語の「単語（たんご）」「句（く）」「文（ぶん）」の違いに似ています。

わかりやすく「<ruby>料理<rt>りょうり</rt></ruby>」に<ruby>例<rt>たと</rt></ruby>えて説明しますね。


### <ruby>演算<rt>えんざん</rt></ruby>子（えんざんし）／ Operator

**→「道具（どうぐ）」のこと**

数字を足したり、比べたりするための「記号（きごう）」です。そのときにも **「料理の道具」のようなもの** と説明したのは、覚えているでしょうか？

* **<ruby>例<rt>れい</rt></ruby>：** `+`, `-`, `*`, `/`, `=`, `>`, `&&`
* **料理でいうと：** <ruby>包丁<rt>ほうちょう</rt></ruby>（ほうちょう）、お<ruby>鍋<rt>なべ</rt></ruby>（なべ）、フライパン

しかし、道具を用意しただけでは何も起きません。道具を使って何かをする必要があります。


### 式（しき）／ Expression

**→「<ruby>計算<rt>けいさん</rt></ruby>のフレーズ」のこと**

<ruby>演算<rt>えんざん</rt></ruby>子（道具）を使って、**「答え（値）」を出すもの**です。

* **<ruby>例<rt>れい</rt></ruby>：** `10 + 5` （答えは15）、`a == b` （答えは「はい」か「いいえ」）
* **料理でいうと：** 「野菜（やさい）を切る」「お肉を焼く」
* 何かのアクションをして、**「切った野菜」という結果**が生まれます。


### 文（ぶん）／ Statement

**→「コンピューターへの <ruby>命令<rt>めいれい</rt></ruby>」のこと**

「〜をしてください」という、**ひとつの<ruby>完璧<rt>かんぺき</rt></ruby>な指示** です。

JavaScriptでは、最後にセミコロン `;` をつけます。

* **<ruby>例<rt>れい</rt></ruby>：** `let x = 10 + 5;` （xという箱に、15を入れてください！）
* **料理でいうと：** 「カレーを作ってください！」
* 道具（<ruby>演算<rt>えんざん</rt></ruby>子）を使い、ステップ（式）を<ruby>組<rt>く</rt></ruby>み<ruby>合<rt>あ</rt></ruby>わせて、**ひとつの仕事（文）**を終わらせます。


### まとめの比較表

| 名前 | 英語 | イメージ | 役割（やくわり） |
| --- | --- | --- | --- |
| **<ruby>演算<rt>えんざん</rt></ruby>子** | Operator | 道具 | `+` や `=` などの記号 |
| **式** | Expression | フレーズ | 何かの* *「値（あたい）」** になるもの |
| **文** | Statement | 命令（文） | コンピューターに**「動作」**をさせるもの |

> [!TIP]
> **<ruby>見分<rt>みわ</rt></ruby>けかたのコツ：**
> 最後に `;`（セミコロン）がついているものは、ほとんどの場合 **「文」** です。日本語の「。」（まる）と同じです。

## II. <ruby>関数<rt>かんすう</rt></ruby>とは何か？

「<ruby>関数<rt>かんすう</rt></ruby>」は、プログラミングの中でとても便利な **「<ruby>魔法<rt>まほう</rt></ruby>の箱」** や **「<ruby>自動販売機<rt>じどうはんばいき</rt></ruby>」** のようなものです。

漢字で見ると<ruby>難<rt>むずか</rt></ruby>しそうですが、その中身は、いくつかの **「文」** をひとつ（ひとつ）にまとめた **「セット」** です。とてもシンプルですよ！ 


**<ruby>関数<rt>かんすう</rt></ruby>は「料理マシン」と同じ！**

<ruby>関数<rt>かんすう</rt></ruby>をイメージするときは、キッチンにある「ジューサー（ミキサー）」を思い出してください。

1. **入れるもの（<ruby>引数<rt>ひきすう</rt></ruby>：ひきすう）：** リンゴを入れる。
2. **中での仕事（処理：しょり）：** リンゴを細かくつぶす。
3. **出てくるもの（<ruby>戻<rt>もど</rt></ruby>り<ruby>値<rt>ち</rt></ruby>：もどりち）：** リンゴジュースが出てくる！

一度この「マシン」を作っておけば、次からは「リンゴ」を入れるだけで、いつでもリンゴジュースを作れます。それだけでなく、「オレンジ」や「バナナ」を入れれば、オレンジジュースやバナナジュースを作ることもできるのです。

こんどは、**「自動販売機（じどうはんぱいき）」** に<ruby>例<rt>たと</rt></ruby>えで見ます。

* お金を入れる・ボタンを押す： これが、<ruby>関数<rt>かんすう</rt></ruby>に<ruby>渡<rt>わた</rt></ruby>すデータ **「<ruby>引数<rt>ひきすう</rt></ruby>（ひきすう）」** です。
* 飲み物が出る： これが、命令を実行した<ruby>結果<rt>けっか</rt></ruby>である **「<ruby>戻<rt>もど</rt></ruby>り<ruby>値<rt>ち</rt></ruby>（もどりち）」** です。

自動販売機の中には、「お金を数える」「<ruby>在庫<rt>ざいこ</rt></ruby>（ざいこ）を確認する」といった複雑（ふくざつ）なプログラムが入っています。でも、私たちは「ボタンを押す」というひとつの命令だけで、欲しい結果を受け取ることができます。

**なぜ「<ruby>関数<rt>かんすう</rt></ruby>」を使うの？**

同じコードを何度も書くのは大変だし、間違いのもとです。<ruby>関数<rt>かんすう</rt></ruby>を使うメリットは2つあります。

* **楽ができます：** 一度作れば、名前を呼ぶだけで何度でも使えます。同じことを何度も書かなくてよくなります。
* **直しやすくなります：** 挨拶の言葉を変えたいとき、1ヶ所（<ruby>関数<rt>かんすう</rt></ruby>の中）を直すだけで全部変わる。
* **整理しやすくなります： 長いコードがスッキリします。

次は、この便利な「<ruby>関数<rt>かんすう</rt></ruby>」をどうやって作るのか、ルールを見ていきましょう。

## III. <ruby>関数<rt>かんすう</rt></ruby>の作り方（<ruby>定義<rt>ていぎ</rt></ruby>：ていぎ）

<ruby>関数<rt>かんすう</rt></ruby>を新しく作ることを「<ruby>定義<rt>ていぎ</rt></ruby>（ていぎ）」と言います。 JavaScriptでは、誰が見ても分かりやすいように、名前の付け方に世界共通のルールがあります。

**基本の<ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>**

以下のキーワードと記号を<ruby>組<rt>く</rt></ruby>み<ruby>合<rt>あ</rt></ruby>わせて作ります。

* function： 「今から<ruby>関数<rt>かんすう</rt></ruby>を作ります」という合図（あいず）です。
* <ruby>関数<rt>かんすう</rt></ruby>名： そのセットに付ける名前です。
* ()： <ruby>引数<rt>ひきすう</rt></ruby>（データ）を受け取るための場所です。
* {}： この中に、実行したい命令をすべて書きます。これを「ブロック」と呼びます。

**良い名前をつけるルール**

プログラマーは、<ruby>関数<rt>かんすう</rt></ruby>が「何をするものか」すぐ分かるように、以下のルールを守ります。

1. 動詞（どうし）から始める： get（取る）、show（見せる）、calculate（計算する）、handle（処理する）などの動詞を使います。
2. キャメルケース（camelCase）： 2つ目以降の単語の最初を大文字にします。<ruby>例<rt>れい</rt></ruby>：showMessage, calculateTotal

**コードの<ruby>例<rt>れい</rt></ruby>**

「挨拶（あいさつ）をしてくれるマシン」を作ってみましょう。

```
// 準備: 関数（かんすう）を定義（ていぎ）する
function sayHello() {
  console.log("こんにちは！JavaScriptの世界へようこそ。");
}
```

これで「準備」ができました。次は、これを目的に合わせて「動かす」方法を説明します。

## V. <ruby>引数<rt>ひきすう</rt></ruby>（ひきすう）の活用と注意点

今度は<ruby>引数<rt>ひきすう</rt></ruby>をとる<ruby>関数<rt>かんすう</rt></ruby>を作ってみましょう。

「名前を呼んで、挨拶（あいさつ）をしてくれるマシン」を作ってみましょう。

```javascript
// 1. 定義（ていぎ）：マシンを作る
function sayHello(name) {
  return "こんにちは、" + name + "さん！";
}
```

* **`function`**： 「これからマシンを作るよ！」という合図。
* **`name`（<ruby>引数<rt>ひきすう</rt></ruby>）**： マシンに入れる材料（ここでは名前）。
* **`return`（<ruby>戻<rt>もど</rt></ruby>り<ruby>値<rt>ち</rt></ruby>）**： マシンが最後に出してくれる結果。

<ruby>関数<rt>かんすう</rt></ruby>に`name`という<ruby>変数<rt>へんすう</rt></ruby>に入ったデータを渡すことで、処理を<ruby>柔軟<rt>じゅうなん</rt></ruby>（じゅうなん）に変えることができます。

では、<ruby>引数<rt>ひきすう</rt></ruby>をとる<ruby>関数<rt>かんすう</rt></ruby>を呼び出すときは、どのようにすればよいでしょうか？

```
// 2. 実行：ボタンを押して動かす
let message = sayHello("田中");
console.log(message); // 「こんにちは、田中さん！」と出る
```

このように、`()`の中に<ruby>関数<rt>かんすう</rt></ruby>に渡すデータを書きます。

ここで、

* 仮<ruby>引数<rt>ひきすう</rt></ruby>（かりひきすう）： <ruby>関数<rt>かんすう</rt></ruby>の中で使う「データの箱」の名前。ここでは`name`
* 実<ruby>引数<rt>ひきすう</rt></ruby>（じつひきすう）： 呼び出すときに実際に渡す「中身」のデータ。ここでは"田中"という文字列

というふうに呼びます。この呼び方もよく使うので、覚えてください。

## VI. <ruby>変数<rt>へんすう</rt></ruby>のスコープ（Scope）：<ruby>変数<rt>へんすう</rt></ruby>の見える範囲

<ruby>変数<rt>へんすう</rt></ruby>は、どこからでも見えるわけではありません。この「見える範囲」をスコープと呼びます。スコープを正しく使うことで、プログラムのミス（バグ）を減らすことができます。

プログラミングの「スコープ（Scope）」は、日本語でいうと **「有効（ゆうこう）な範囲（はんい）」**、つまり **「その<ruby>変数<rt>へんすう</rt></ruby>がどこまで使えるか」というルールのこと** です。

### グローバルスコープ (Global Scope)

**→「外の世界（公園やインターネット）」のようなもの**

プログラムのどこからでも見える、一番広いスコープです。

```javascript
let world = "こんにちは世界！"; // グローバル変数（へんすう）

function sayWorld() {
  console.log(world); // 関数（かんすう）の中でも使える
}

sayWorld();
console.log(world); // 関数（かんすう）の外でも使える

```

* **イメージ：** 空に浮かんでいる「お月様（つきさま）」です。どこにいても見えますよね。


### ローカルスコープ / <ruby>関数<rt>かんすう</rt></ruby>スコープ (Local Scope)

**→「家の中」のようなもの**

`function`（<ruby>関数<rt>かんすう</rt></ruby>）の中で作った<ruby>変数<rt>へんすう</rt></ruby>は、その<ruby>関数<rt>かんすう</rt></ruby>の**外からは見えません。**

```javascript
function myHouse() {
  let secret = "これは秘密（ひみつ）です"; // ローカル変数（へんすう）
  console.log(secret); // 関数（かんすう）の中ならOK
}

myHouse();
console.log(secret); // ❌ エラー！外からは見えません。

```

* **イメージ：** 家の中にある「テレビのりもこん」です。隣の家の人（<ruby>関数<rt>かんすう</rt></ruby>の外）は使えません。

***

### ブロックスコープ (Block Scope)

**→「小さな箱（はこ）」のようなもの**

`if` 文や `for` 文などの `{ }` （波（なみ）かっこ）の中で `let` や `const` を使って作った<ruby>変数<rt>へんすう</rt></ruby>は、その **`{ }` の中だけでしか使えません。**

```javascript
if (true) {
  let fruit = "りんご"; // ブロック変数（へんすう）
  console.log(fruit); // ブロックの中ならOK
}

console.log(fruit); // ❌ エラー！箱の外には出せません。

```

* **イメージ：** ロッカーの中の荷物です。ロッカー（`{ }`）を閉めたら、外からは触れません。


### ポイントまとめ

| スコープの名前 | 使える場所 | わかりやすい<ruby>例<rt>たと</rt></ruby>え |
| --- | --- | --- |
| **グローバル** | どこでもOK | 公園にある時計（みんな見れる） |
| **ローカル（<ruby>関数<rt>かんすう</rt></ruby>）** | その<ruby>関数<rt>かんすう</rt></ruby>の中だけ | あなたの部屋の電気（外からは消せない） |
| **ブロック** | `{ }` の中だけ | お弁当箱の中のおかず（外にはこぼれない） |


### なぜこれが大事なの？

もし、すべての<ruby>変数<rt>へんすう</rt></ruby>を「グローバル（どこでも使える）」にすると、知らないうちに別の場所で**データが<ruby>上書<rt>うわが</rt></ruby>き（書きかえ）**されてしまい、バグの原因になります。

> [!TIP]
> **「できるだけ狭（せま）く」が基本！**
> 留学生には、「<ruby>変数<rt>へんすう</rt></ruby>はなるべく小さな箱（ブロックスコープ）に入れて、必要なときだけ外に出そうね」と教えてあげてください。


**var は使わない、const を使おう**

JavaScriptには<ruby>変数<rt>へんすう</rt></ruby>の作り方が3つありますが、安全性のために<ruby>使<rt>つか</rt></ruby>い<ruby>分<rt>わ</rt></ruby>けが重要です。

`var`・`let`・`const` の違いは、プログラミングの世界では **「古いルール」と「新しいルール」の違い** だと考えると分かりやすいです。

むかしは `var` しかありませんでしたが、今はもっと安全な `let` と `const` を使うのが普通です。


### 違いをまとめた表

まずは、この表で全体（ぜんたい）のルールを見てみましょう。

| 種類 | スコープ（範囲） | 値の<ruby>書<rt>か</rt></ruby>き<ruby>換<rt>かえ</rt></ruby>え | 同じ名前で作り直し |
| --- | --- | --- | --- |
| **`var`** | **<ruby>関数<rt>かんすう</rt></ruby>**スコープ（広い） | OK | OK（危ない！） |
| **`let`** | **ブロック**スコープ（狭い） | OK | ❌ だめ |
| **`const`** | **ブロック**スコープ（狭い） | ❌ だめ | ❌ だめ |


### なぜ `var` は使わないほうがいいの？

`var` は「ルールがゆるすぎる」ので、ミスが起きやすいんです。

#### ① ブロック `{ }` を無視して外に出てしまう

`var` は `if` や `for` の外まで飛び出してしまいます。

```javascript
if (true) {
  var leak = "外に出ちゃうよ！";
}
console.log(leak); // ⭕ 表示（ひょうじ）できてしまう（これがバグの原因になる）

if (true) {
  let safe = "外には出ないよ";
}
console.log(safe); // ❌ エラー（外からは見えないので安全！）

```

#### ② 同じ名前で何度でも作れてしまう

`var` は、同じ名前の<ruby>変数<rt>へんすう</rt></ruby>をうっかり2回作っても怒られません。

```javascript
var name = "田中";
var name = "鈴木"; // ⭕ OK（前の「田中」が消えてしまう！）

let age = 20;
let age = 25; // ❌ エラー！「もうありますよ！」と教えてくれる

```

### `let` と `const` はどう<ruby>使<rt>つか</rt></ruby>い<ruby>分<rt>わ</rt></ruby>ける？

今のプログラミングでは、この **「<ruby>使<rt>つか</rt></ruby>い<ruby>分<rt>わ</rt></ruby>け」** がとても大切です。<ruby>繰<rt>く</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>しになりますが、ここでも確認しておきましょう。

#### `const`（定数：ていすう）

* **「カギをかけた箱」** です。
* 一度入れたら、中身を変えることはできません。
* **基本はこれを使います。** 途中で値が変わらないほうが、読む人が安心できるからです。

#### `let`（<ruby>変数<rt>へんすう</rt></ruby>：へんすう）

* **「カギのない箱」** です。
* 中身を何度も<ruby>入<rt>い</rt></ruby>れ<ruby>替<rt>かえ</rt></ruby>ることができます。
* `for` 文のカウンター（`i++` など）のように、**数字が変わるものだけ** に使います。

### 学生へのアドバイス

1. **まずは `const` を使おう！**（一番安全だから）
2. もし、あとで値を<ruby>書<rt>か</rt></ruby>き<ruby>換<rt>かえ</rt></ruby>える必要があるなら **`let` に変えよう。**
3. **`var` はもう使わない！**（古い本や古いサイトには書いてあるけど、今は使わないのがルールだよ）

> [!TIP]
> **「定数（const）」** の **「定」** は「決定（けってい）」の「定」。
> つまり、「もう決まっていて動かない」という意味だと教えると分かりやすいですよ。

## VII. 無名<ruby>関数<rt>かんすう</rt></ruby>（むめいかんすう）

ここまで「<ruby>関数<rt>かんすう</rt></ruby>（マシン）」についてお<ruby>話<rt>はなし</rt></ruby>ししました。次は、そのマシンの **「ちょっと特別な形」** である、**無名<ruby>関数<rt>かんすう</rt></ruby>（むめいかんすう）とアロー<ruby>関数<rt>かんすう</rt></ruby>** について説明します。

どちらも、今のJavaScriptでは毎日使うとても大切な<ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>です！

***

### 無名<ruby>関数<rt>かんすう</rt></ruby>（むめいかんすう）

**→「名前のない マシン」のこと**

普通の<ruby>関数<rt>かんすう</rt></ruby>は `function sayHello() ...` のように名前をつけますが、名前をつけないで作ることもできます。

* **なぜ名前をつけないの？**
「その場ですぐに使うだけだから、名前をつけるのが面倒（めんどう）」という時に使います。

* **<ruby>例<rt>たと</rt></ruby>え話：**
名前のある<ruby>関数<rt>かんすう</rt></ruby>が「名前を<ruby>登録<rt>とうろく</rt></ruby>（とうろく）した専属のシェフ」なら、無名<ruby>関数<rt>かんすう</rt></ruby>は「その時だけ手伝ってくれる助っ人（すけっと）」のようなイメージです。

```javascript
// 普通の関数（かんすう）（名前がある）
function sayHi() {
  console.log("こんにちは！");
}

// 無名関数（かんすう）（名前がない）
// 変数（へんすう）に入れて使うことが多いです
const sayHi2 = function() {
  console.log("こんにちは！");
};

```


### アロー<ruby>関数<rt>かんすう</rt></ruby> (Arrow Function)

**→「もっと短く書ける、<ruby>最新<rt>さいしん</rt></ruby>のショートカット」のこと**

「無名<ruby>関数<rt>かんすう</rt></ruby>」をさらに短く、かっこよく書く方法です。 **`=>`（矢印：アロー）** を使うので「アロー<ruby>関数<rt>かんすう</rt></ruby>」と呼びます。

#### <ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>の変化を見てみよう

どれも同じ<ruby>動<rt>うご</rt></ruby>きをしますが、下に行くほど短くなります。

1. **普通の<ruby>関数<rt>かんすう</rt></ruby>：**
`function(a, b) { return a + b; }`
2. **アロー<ruby>関数<rt>かんすう</rt></ruby>：**
`(a, b) => { return a + b; }`
3. **もっと短く（1行のとき）：**
`(a, b) => a + b;`

***

### どうしてアロー<ruby>関数<rt>かんすう</rt></ruby>を使うの？

留学生のみなさんには、この3つのメリットを覚えてほしいです。

* **タイピングが楽：** `function` と書かなくていいので、指が疲れません。
* **<ruby>見<rt>み</rt></ruby>た<ruby>目<rt>め</rt></ruby>がスッキリ：** コードが短くなって、読みやすくなります。
* **「this」のルールがやさしい：** （少し<ruby>難<rt>むずか</rt></ruby>しい話ですが）アロー<ruby>関数<rt>かんすう</rt></ruby>を使うと、プログラムが「自分（this）」を見失いにくくなり、エラーが減ります。

***

### どんな時に使うの？（具体的な<ruby>例<rt>れい</rt></ruby>）

一番よく使うのは、**「他の<ruby>関数<rt>かんすう</rt></ruby>の中で、ちょっと手伝ってほしい時」** です。

```javascript
// 3秒後に「時間ですよ！」と表示（ひょうじ）する
setTimeout(() => {
  console.log("時間ですよ！");
}, 3000);

```

このように、`setTimeout` という<ruby>関数<rt>かんすう</rt></ruby>の中に、さらに小さな<ruby>関数<rt>かんすう</rt></ruby>（アロー<ruby>関数<rt>かんすう</rt></ruby>）を入れて使います。


### 学生へのアドバイス

| 名前 | <ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>の特徴 | ニュアンス |
| --- | --- | --- |
| **普通の<ruby>関数<rt>かんすう</rt></ruby>** | `function 名前() { ... }` | どこでも使える、基本の形 |
| **無名<ruby>関数<rt>かんすう</rt></ruby>** | `function() { ... }` | 名前をつけるのが面倒なとき |
| **アロー<ruby>関数<rt>かんすう</rt></ruby>** | `() => { ... }` | **今、一番よく使う形！** かっこよくて便利 |

> [!TIP]
> **<ruby>初心者<rt>しょしんしゃ</rt></ruby>のうちは…**
> 自分で書くときは、まず **「アロー<ruby>関数<rt>かんすう</rt></ruby>」** を練習してみてください。今の現場では 90% くらいアロー<ruby>関数<rt>かんすう</rt></ruby>が使われています。もし古いコードを読んで `function` が出てきたら、「あ、これはお父さん世代の<ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>だな」と思えばOKです！

いかがでしょうか？「矢印（`=>`）が出てきたら、短い<ruby>関数<rt>かんすう</rt></ruby>の合図だ！」と覚えると簡単ですよ。

## VIII. まとめ

JavaScriptの「<ruby>関数<rt>かんすう</rt></ruby>」をマスターするために、5つのポイントを<ruby>振<rt>ふ</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>りましょう。

* <ruby>定義<rt>ていぎ</rt></ruby>： function を使い、動詞から始まる名前をつけてセットにしましょう。
* 呼び出し： <ruby>関数<rt>かんすう</rt></ruby>名のあとの () を忘れずにつけて実行しましょう。
* スコープ： <ruby>変数<rt>へんすう</rt></ruby>は const を使い、なるべく {} の中で安全に管理しましょう。
* 無名<ruby>関数<rt>かんすう</rt></ruby>： 一度きりの処理は、名前をつけずにスマートに書きましょう。

So What?（だから何？） <ruby>関数<rt>かんすう</rt></ruby>を学ぶことは、ただコードを書くことではありません。大きな問題（プログラム）を、小さくて扱いやすい「<ruby>関数<rt>かんすう</rt></ruby>のセット」に分けて考えることです。これが**「プログラマーのように考える」**ための第<ruby>一歩<rt>いっぽ</rt></ruby>です。


### 留学生向けの言葉のガイド

<ruby>難<rt>むずか</rt></ruby>しい専門用語を、もっと簡単な日本語に言いかえてみましょう。

| 専門用語 | やさしい言いかえ | 英語 |
| --- | --- | --- |
| **<ruby>定義<rt>ていぎ</rt></ruby>（ていぎ）する** | マシンを**作る** | Define |
| **呼び出す / 実行する** | マシンを**動かす** | Call / Execute |
| **<ruby>引数<rt>ひきすう</rt></ruby>（ひきすう）** | **入れるもの** / 材料 | Arguments / Parameters |
| **<ruby>戻<rt>もど</rt></ruby>り<ruby>値<rt>ち</rt></ruby>（もどりち）** | **返ってくるもの** / 答え | Return value |

> [!TIP]
> **「<ruby>引数<rt>ひきすう</rt></ruby>（ひきすう）」の覚え方**
> 「いんすう」と読みたくなりますが、プログラミングでは **「ひきすう」** と読みます。「（材料を）引き連れてくる数」だと覚えるといいですよ！
