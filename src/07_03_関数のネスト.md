# <ruby>関数<rt>かんすう</rt></ruby>のネスト

<ruby>関数<rt>かんすう</rt></ruby>の中に、さらに<ruby>関数<rt>かんすう</rt></ruby>を書くことができます。これを **「ネスト（入れ子）」** と呼びます。これは、プログラムを整理して情報を守る **「カプセル化（情報の保護）」** に役立ちます。

<ruby>親子<rt>おやこ</rt></ruby>関係とスコープ

* ルール： <ruby>内側<rt>うちがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>（子供）は、<ruby>外側<rt>そとがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>（親）にある<ruby>変数<rt>へんすう</rt></ruby>を見ることができます。
* <ruby>例<rt>たと</rt></ruby>え： 「子供は自分の家の冷蔵庫（親の<ruby>変数<rt>へんすう</rt></ruby>）を開けて中身を見ることができます。でも、外にいる隣の人（外の<ruby>関数<rt>かんすう</rt></ruby>）は、その家の中を見ることはできません。」

```JavaScript
function outerFunc() {
  var a;
  function innerFunc() {
    var b;
  }
}
```
このように<ruby>構造<rt>こうぞう</rt></ruby>を作ることで、大切なデータを外から勝手に触られないように守ることができます。


***

## I. <ruby>関数<rt>かんすう</rt></ruby>のネスト（入れ子構造）

プログラミングをしていると、<ruby>関数<rt>かんすう</rt></ruby>の中にさらに別の<ruby>関数<rt>かんすう</rt></ruby>を書くことがあります。これを **「ネスト（入れ子：いれこ）」** と言います。

これに伴う「スコープ（見える範囲）」のルールは、留学生のみなさんには **「家と部屋」** に<ruby>例<rt>たと</rt></ruby>えて説明すると、とても分かりやすくなりますよ！

***

### 1. ネスト（入れ子）ってなに？

マトリョーシカ（ロシアの人形）のように、大きな箱の中に小さな箱が入っている状態のことです。

JavaScriptでは、このように書きます。

```javascript
function outer() { // 外側（そとがわ）の関数（かんすう）（大きな箱）
  
  function inner() { // 内側（うちがわ）の関数（かんすう）（小さな箱）
    // 何かの処理
  }

}

```



### 2. スコープの絶対ルール（家と部屋のルール）

ネストされた<ruby>関数<rt>かんすう</rt></ruby>のスコープには、大切な **2つのルール** があります。

#### ルール①：中は「外」が見える

自分の部屋（<ruby>内側<rt>うちがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>）にいても、窓から外（<ruby>外側<rt>そとがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>）の景色は見えますよね。つまり、**<ruby>内側<rt>うちがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>から、<ruby>外側<rt>そとがわ</rt></ruby>の<ruby>変数<rt>へんすう</rt></ruby>は使えます。**

#### ルール②：外は「中」が見えない

家の外（<ruby>外側<rt>そとがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>）にいる人は、あなたの部屋（<ruby>内側<rt>うちがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>）の引き出しの中身は見えません。つまり、**<ruby>外側<rt>そとがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>から、<ruby>内側<rt>うちがわ</rt></ruby>の<ruby>変数<rt>へんすう</rt></ruby>は使えません。**



### 3. 具体的なコードで見てみよう！

このルールを実際のコードで確認してみましょう。

```javascript
function grandParent() {
  let money = "1000円"; // おじいさんのお金

  function child() {
    let candy = "アメ玉"; // 子供のお菓子
    
    // ✅ ルール①：中は外が見える
    console.log(money); // 「1000円」が使える！
  }

  child();

  // ❌ ルール②：外は中が見えない
  console.log(candy); // エラー！子供の「アメ玉」は外からは見えません。
}

grandParent();

```



### 4. なぜ「ネスト」を使うの？

「わざわざ<ruby>関数<rt>かんすう</rt></ruby>の中に<ruby>関数<rt>かんすう</rt></ruby>を書かなくてもいいのでは？」と思うかもしれません。でも、これには大きなメリットがあります。

* **秘密（ひみつ）を守る：** `inner`<ruby>関数<rt>かんすう</rt></ruby>は、`outer`<ruby>関数<rt>かんすう</rt></ruby>の中だけで使う「<ruby>専用<rt>せんよう</rt></ruby>の道具」になります。他の場所から勝手に使われないように隠すことができるのです。
* **整理<ruby>整頓<rt>せいとん</rt></ruby>（せいりせいとん）：** 関連する機能をまとめておくことで、コードが読みやすくなります。


### 留学生へのまとめ

| <ruby>視点<rt>してん</rt></ruby>（目線） | 見える？ | 理由（たとえ） |
| --- | --- | --- |
| **<ruby>内側<rt>うちがわ</rt></ruby>から → <ruby>外側<rt>そとがわ</rt></ruby>へ** | **見える！ (OK)** | 部屋の窓から外が見えるから。 |
| **<ruby>外側<rt>そとがわ</rt></ruby>から → <ruby>内側<rt>うちがわ</rt></ruby>へ** | **見えない！ (NG)** | 外から他人の部屋の中は見えないから。 |

> [!TIP]
> **「スコープチェーン」という言葉**
> もし学校で「スコープチェーン」という言葉が出てきたら、それは**「自分に無ければ、外の箱に探しに行く」**というルールのことだと思ってください。
> 1. 自分の部屋にある？（なければ次へ）
> 2. リビング（外の<ruby>関数<rt>かんすう</rt></ruby>）にある？（なければ次へ）
> 3. お庭（グローバル）にある？
> 
> 
> こうやって順番に探しに行くのが、ネストの面白い<ruby>仕組<rt>しく</rt></ruby>みです！



いかがでしょうか？「外からは見えないけれど、中からは見える」というルールが分かれば、ネストはもう怖くありません。

## II. クロージャ

**クロージャ (Closure)** は、日本語でいうと **「<ruby>記憶<rt>きおく</rt></ruby>（きおく）を持っている<ruby>関数<rt>かんすう</rt></ruby>」** のことです。

少し<ruby>難<rt>むずか</rt></ruby>しい<ruby>概念<rt>がいねん</rt></ruby>ですが、**「<ruby>魔法<rt>まほう</rt></ruby>のリュックサック」** をイメージすると、とても分かりやすくなります。

### 1. クロージャをイメージで理解しよう！

普通の<ruby>関数<rt>かんすう</rt></ruby>は、仕事が終わると、その中で使っていた<ruby>変数<rt>へんすう</rt></ruby>のことをすべて忘れてしまいます。
しかし、クロージャは違います。

* **普通の<ruby>関数<rt>かんすう</rt></ruby>：** 仕事が終わったら、机の上をきれいに片付ける（<ruby>変数<rt>へんすう</rt></ruby>が消える）。
* **クロージャ：** 仕事が終わっても、大切な<ruby>変数<rt>へんすう</rt></ruby>を**「リュックサック（スコープ）」**に入れて持ち歩く。次に呼ばれたとき、そのリュックの中から<ruby>前回<rt>ぜんかい</rt></ruby>の続きを取り出せる。

***

### 2. 具体的なコードで見てみよう

一番わかりやすい<ruby>例<rt>れい</rt></ruby>は、数字を数える「カウンター」です。

```javascript
function createCounter() {
  let count = 0; // これが「リュックの中身（秘密の変数（へんすう））」

  return function() { 
    count++; // 外側（そとがわ）の変数（へんすう）を覚えている！
    console.log(count + "回目です");
  };
}

const myCounter = createCounter(); // ここで関数（かんすう）（マシン）を受け取る

myCounter(); // 「1回目です」
myCounter(); // 「2回目です」
myCounter(); // 「3回目です」

```

#### ここが「<ruby>魔法<rt>まほう</rt></ruby>」のポイント：

普通、`createCounter` という<ruby>関数<rt>かんすう</rt></ruby>が動いた後、その中の `count = 0` という<ruby>変数<rt>へんすう</rt></ruby>は消えてしまうはずです。
でも、戻ってきた「<ruby>内側<rt>うちがわ</rt></ruby>の<ruby>関数<rt>かんすう</rt></ruby>」が `count` を**リュックに入れて持ち続けている**ので、ずっと数字が増えていくのです。

***

### 3. なぜクロージャを使うの？

留学生のみなさんには、この2つのメリットを伝えてあげてください。

1. **データを隠せる（プライバシー）：**
`count` という<ruby>変数<rt>へんすう</rt></ruby>は<ruby>関数<rt>かんすう</rt></ruby>の外からは絶対に見えません。外から勝手に数字を変えられたくない「大切なデータ」を守るのに便利です。
2. **<ruby>専用<rt>せんよう</rt></ruby>のマシンが作れる：**
`const counterA = createCounter();`
`const counterB = createCounter();`
と書けば、それぞれが **別々のリュック（<ruby>記憶<rt>きおく</rt></ruby>）** を持つことができます。AとBの数字は混ざりません。

***

### 4. 留学生向けの言葉のガイド

| <ruby>難<rt>むずか</rt></ruby>しい言葉 | やさしい言いかえ | 英語 |
| --- | --- | --- |
| **<ruby>保持<rt>ほじ</rt></ruby>する** | **ずっと持っている** | Hold / Keep |
| **カプセル化** | **中身を隠して守る** | Encapsulation |
| **<ruby>参照<rt>さんしょう</rt></ruby>する** | **見に行く / 使う** | Reference |

***

> [!TIP]
> **「閉じる」という意味**
> クロージャの「クローズ（Close）」は、**「<ruby>変数<rt>へんすう</rt></ruby>を<ruby>関数<rt>かんすう</rt></ruby>の中に閉じ込める（閉じ込めて守る）」** という意味から来ています。
> 「自分だけの秘密のポケットを持っている<ruby>関数<rt>かんすう</rt></ruby>なんだな」と思えばバッチリです！

いかがでしょうか？「<ruby>関数<rt>かんすう</rt></ruby>が<ruby>変数<rt>へんすう</rt></ruby>をリュックに入れて持ち歩いている」というイメージが伝われば、クロージャの理解はぐっと深まります。
