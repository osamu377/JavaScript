# 書いてみよう

## I. 初めてのプログラム：Hello World! とポップアップ

プログラミングの第<ruby>一歩<rt>いっぽ</rt></ruby>は、自分の指示がブラウザに伝わったことを確認することです。<ruby>画面<rt>がめん</rt></ruby>にメッセージを出す2つの方法を学びましょう。

### alert()：ユーザーへの通知

ブラウザに小さなウィンドウを出して、ユーザーに大切なメッセージを伝えます。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Hello</title>
</head>
<body>
    <p>Let's JavaScript --<br>
        <script type="text/javascript">
          alert("こんにちは！");
        </script>
    </p>
</body>
</html>
```

これをブラウザで見ると、このようになります。<a href="./html/02_02_alert.html">ここをクリックしてみましょう。</a> 

### console.log()：<ruby>開発<rt>かいはつ</rt></ruby>者のための確認（デバッグ）

「デバッグ（プログラムのミスを見つけて直すこと）」のために使います。ユーザーには見えない、<ruby>開発<rt>かいはつ</rt></ruby>者のための「確認用メッセージ」をコンソールに<ruby>出力<rt>しゅつりょく</rt></ruby>します。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Hello</title>
</head>
<body>
    <p>Let's JavaScript --<br>
        <script type="text/javascript">
          console.log("プログラムがここまで動（うご）きました");
        </script>
    </p>
</body>
</html>
```

#### 「コンソール」の開き方

自分の書いた「<ruby>意図<rt>いと</rt></ruby>（やりたいこと）」がブラウザに正しく伝わったかどうかは、ここで確認します。

|OS|操作方法|
|---|---|
|Windows|F12 キー（ノートPCは Fn + F12） または Ctrl + Shift + I|
|Mac|Command + Option + I|

コンソールを開いて、<a href="./html/02_03_console_log.html">ここをクリックしてみましょう。</a>

自分の書いた1行のコードでブラウザが<ruby>反応<rt>はんのう</rt></ruby>したとき、その「喜び」がこれからの学習の力になります！

### 文（ぶん）／ Statement

`alert("こんにちは！");` のような **「コンピューターへの <ruby>命令<rt>めいれい</rt></ruby>」** を **「文」** と呼びます。「〜をしてください」という、**ひとつの<ruby>完璧<rt>かんぺき</rt></ruby>な指示**です。

JavaScriptでは、最後にセミコロン `;` をつけます。

## II. イベントとイベントハンドラ：<ruby>動<rt>うご</rt></ruby>きを作るきっかけ

JavaScriptは、ユーザーの「クリック」などの操作に<ruby>反応<rt>はんのう</rt></ruby>して<ruby>動<rt>うご</rt></ruby>き出します。この「きっかけ」を理解すると、Webサイトに命を<ruby>吹<rt>ふ</rt></ruby>き<ruby>込<rt>こ</rt></ruby>むことができます。

### スイッチと電球の関係

* イベント（操作）： ユーザーがボタンを押す「スイッチ」です。
* ハンドラ（処理）： スイッチが入ったときに実行される「電球がつく」という<ruby>動<rt>うご</rt></ruby>きです。

よく使われるイベントハンドラ（きっかけ）を整理しました。

|カテゴリ|イベント名|きっかけ（タイミング）|使用<ruby>例<rt>れい</rt></ruby>|
|---|---|---|---|
|マウス|onclick|クリックされたとき|ボタンを押して計算を始める|
|マウス|onmouseover|マウスが上に乗ったとき|<ruby>画像<rt>がぞう</rt></ruby>の色を変える|
|ページ|onload|ページを<ruby>読<rt>よ</rt></ruby>み<ruby>込<rt>こ</rt></ruby>み終わったとき|「ようこそ」と挨拶を出す|
|フォーム|oninput|文字を<ruby>入力<rt>にゅうりょく</rt></ruby>したとき|リアルタイムで文字数を数える|
|フォーム|onfocus|<ruby>入力<rt>にゅうりょく</rt></ruby>欄を選択したとき|<ruby>入力<rt>にゅうりょく</rt></ruby>欄の色を明るくする|

<ruby>記述<rt>きじゅつ</rt></ruby><ruby>例<rt>れい</rt></ruby>： HTMLのボタンに「クリックされたらアラートを出す」<ruby>魔法<rt>まほう</rt></ruby>をかけるときは、こう書きます。

```html
<button onclick="alert('ボタンが押されました！')">クリックしてね</button>
```
これをブラウザで読（よ）み込（こ）むと、以下のようになります。

<button onclick="alert('ボタンが押されました！')">クリックしてみましょう</button>

## III. JavaScriptの呼び出し方と<ruby>外部<rt>がいぶ</rt></ruby>ファイル化

コードが増えてくると、書き場所が重要になります。整理<ruby>整頓<rt>せいとん</rt></ruby>されたコードは、プロの世界の「マナー」です。

1. <ruby>内部<rt>ないぶ</rt></ruby>JavaScript： HTMLの中に `<script>` タグで直接書く方法です。
2. <ruby>外部<rt>がいぶ</rt></ruby>JavaScript： .js という別のファイルに分ける方法です。

**<ruby>外部<rt>がいぶ</rt></ruby>ファイル化のメリット（<ruby>戦略<rt>せんりゃく</rt></ruby>的意義）**

プロの<ruby>開発<rt>かいはつ</rt></ruby>では、必ず「<ruby>外部<rt>がいぶ</rt></ruby>ファイル」を使います。理由は、**「再利用性（ひとつのファイルを多くのページで使い回せること）」** が高まり、修正も一箇所で済むためです。

**<ruby>読<rt>よ</rt></ruby>み<ruby>込<rt>こ</rt></ruby>みの<ruby>手順<rt>てじゅん</rt></ruby>**

* .js ファイル（<ruby>例<rt>れい</rt></ruby>：script.js）を作成し、JavaScriptだけを書きます。
* HTMLの `<body>` タグの最後（閉じタグ `</body>` の直前） に<ruby>読<rt>よ</rt></ruby>み<ruby>込<rt>こ</rt></ruby>みコードを書きます。

なぜ `<body>` の最後に書くの？ ブラウザは上から順番に<ruby>読<rt>よ</rt></ruby>み<ruby>込<rt>こ</rt></ruby>みます。プログラムを最後に<ruby>読<rt>よ</rt></ruby>み<ruby>込<rt>こ</rt></ruby>ませることで、ページの文字や<ruby>画像<rt>がぞう</rt></ruby>を先に出し、ユーザーを待たせない「スムーズな<ruby>表示<rt>ひょうじ</rt></ruby>」が可能になります。
