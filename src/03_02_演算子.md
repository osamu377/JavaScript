# <ruby>演算<rt>えんざん</rt></ruby>子

**JavaScriptの「<ruby>演算<rt>えんざん</rt></ruby>子（えんざんし）」：プログラムに計算や判断をさせる道具**


***


## I. はじめに：<ruby>演算<rt>えんざん</rt></ruby>子（えんざんし）とは何か

プログラミングを学ぶとき、一番大切な道具が「<ruby>演算<rt>えんざん</rt></ruby>子（えんざんし）」です。これは、データを使って計算をしたり、正しいか間違いかを調べたりするための「記号」のことです。

<ruby>演算<rt>えんざん</rt></ruby>子を「料理の道具」に<ruby>例<rt>たと</rt></ruby>えてみましょう。「料理の材料」はデータです。材料を<ruby>包丁<rt>ほうちょう</rt></ruby>で切ったり、鍋で煮たりするように、<ruby>演算<rt>えんざん</rt></ruby>子を使ってデータを<ruby>加工<rt>かこう</rt></ruby>します。また、<ruby>演算<rt>えんざん</rt></ruby>子は「計算機のボタン」にも似ています。ボタンを押すことで、計算機に「何をすべきか」を伝えます。

<ruby>演算<rt>えんざん</rt></ruby>子がないプログラムは、ただの「動かない文字の集まり」です。 <ruby>演算<rt>えんざん</rt></ruby>子を使うことで初めて、プログラムに「知能（判断力）」や「<ruby>動<rt>うご</rt></ruby>き」という「心」が生まれます。

ではまず、<ruby>演算<rt>えんざん</rt></ruby>子の中でもプログラムの<ruby>動<rt>うご</rt></ruby>きを作る基本中の基本、「計算の<ruby>演算<rt>えんざん</rt></ruby>子」から学んでいきましょう。


***


## II. 計算の<ruby>演算<rt>えんざん</rt></ruby>子（<ruby>算術<rt>さんじゅつ</rt></ruby><ruby>演算<rt>えんざん</rt></ruby>子：さんじゅつえんざんし）

<ruby>数値<rt>すうち</rt></ruby>を計算するときに使う記号です。算数のルールと似ていますが、<ruby>掛<rt>か</rt></ruby>け<ruby>算<rt>ざん</rt></ruby>や<ruby>割<rt>わ</rt></ruby>り<ruby>算<rt>ざん</rt></ruby>の記号がプログラミング<ruby>特有<rt>とくゆう</rt></ruby>のものになっています。

### 1. 基本的な計算の記号

プログラムでは、計算の結果を<ruby>画面<rt>がめん</rt></ruby>に出したり、<ruby>変数<rt>へんすう</rt></ruby>（箱）に保存して次の処理に使ったりします。

|<ruby>演算<rt>えんざん</rt></ruby>子|意味|<ruby>例<rt>れい</rt></ruby>|結果|
|---|---|---|---|
|+|<ruby>足<rt>た</rt></ruby>し<ruby>算<rt>ざん</rt></ruby>（<ruby>加算<rt>かさん</rt></ruby>）|5 + 3|8|
|-|<ruby>引<rt>ひ</rt></ruby>き<ruby>算<rt>ざん</rt></ruby>（<ruby>減算<rt>げんさん</rt></ruby>）|10 - 4|6|
|*|<ruby>掛<rt>か</rt></ruby>け<ruby>算<rt>ざん</rt></ruby>（<ruby>乗算<rt>じょうさん</rt></ruby>）|6 * 2|12|
|/|<ruby>割<rt>わ</rt></ruby>り<ruby>算<rt>ざん</rt></ruby>（<ruby>除算<rt>じょさん</rt></ruby>）|10 / 2|5|
|%|余り（剰余）|10 % 3|1（10÷3の余り）|

### 2. 数を1つずつ増やす・減らす

「あと何回繰り返すか」を数えるときなど、ループ処理で非常によく使う特別な記号があります。

* インクリメント（++）: 値を「1」増やします。（<ruby>例<rt>れい</rt></ruby>：count++）
* デクリメント（--）: 値を「1」減らします。（<ruby>例<rt>れい</rt></ruby>：count--）

### 3. 文字列の<ruby>結合<rt>けつごう</rt></ruby>と「Number()」の<ruby>魔法<rt>まほう</rt></ruby>

`+` <ruby>演算<rt>えんざん</rt></ruby>子は文字をつなげることもできますが、注意が必要です。

* "こんにちは" + "田中さん" → "こんにちは田中さん"
* "100" + 1 → "1001" （文字の100と数字の1がくっついてしまう）

このように、文字として数字を扱ってしまうと計算ミス（バグ）になります。そこで便利なのが `Number()` という<ruby>関数<rt>かんすう</rt></ruby>です。これは、**「文字の "100" を<ruby>数値<rt>すうち</rt></ruby>の 100 に変える<ruby>魔法<rt>まほう</rt></ruby>」** です。

* Number("100") + 1 → 101 （正しく計算できます！）

※ <ruby>関数<rt>かんすう</rt></ruby>については、後の章で詳しく説明します。今は、ひとつの **「<ruby>魔法<rt>まほう</rt></ruby>」** だと思っていてください。

計算ができるようになったら、次はプログラムに「はい」か「いいえ」を判断させる方法を見てみましょう。


### 4. 式（しき）／ Expression

`5 + 3` のように、<ruby>演算<rt>えんざん</rt></ruby>子（道具）を使って、**「答え（値）」を出すもの** を **式** と呼びます。「<ruby>計算<rt>けいさん</rt></ruby>のフレーズ」のことです。

* **<ruby>例<rt>れい</rt></ruby>：** `10 + 5` （答えは15）、`a == b` （答えは「はい」か「いいえ」）
* **料理でいうと：** 「野菜（やさい）を切る」「お肉を焼く」
* 何かのアクションをして、**「切った野菜」という結果**が生まれます。

## III. 条件式で使う<ruby>演算<rt>えんざん</rt></ruby>子（比較<ruby>演算<rt>えんざん</rt></ruby>子：ひかくんざんし）

プログラムに「もし〜なら」という判断をさせるためには、2つの値を比べる必要があります。このときに使うのが比較<ruby>演算<rt>えんざん</rt></ruby>子です。これはプログラムが自分で考えて動くための「心臓部」です。

### 1. 値の大きさを比べる

比較した結果は、正しいなら true（<ruby>真<rt>しん</rt></ruby>）、間違いなら false（<ruby>偽<rt>ぎ</rt></ruby>） という「ブール値」で返されます。

* `>` ：左の方が大きい
* `<` ：右の方が大きい
* `>=` ：左の方が右と同じか、大きい（以上）
* `<=` ：左の方が右と同じか、小さい（以下）

### 2. 「同じ」かどうかを調べる（ここは一番大切です！）

「同じ」を調べるとき、JavaScriptには2つの<ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>がありますが、初心者は必ず `===` を使ってください。

* `==`（等しい）: 値が同じか「<ruby>大<rt>おお</rt></ruby>まかに」調べます。
* `===`（<ruby>厳密<rt>げんみつ</rt></ruby>に等しい）: 値だけでなく「データの種類（数字か文字か）」まで正確に調べます。

なぜ `===` がいいのか？ `==` を使うと、JavaScriptが、たのまれてもいないのに、勝手に手伝おうとして、データの種類を変えて比較してしまいます。<ruby>例<rt>たと</rt></ruby>えば、`5 == "5"` は true になってしまいます。このようにJavaScriptが勝手に型を変えてしまうことが、<ruby>予想<rt>よそう</rt></ruby>外のバグの原因になります。正確なプログラムを作るために、型までチェックする `===` を使いましょう。

次は、複数の条件を<ruby>組<rt>く</rt></ruby>み<ruby>合<rt>あ</rt></ruby>わせる「<ruby>論理<rt>ろんり</rt></ruby><ruby>演算<rt>えんざん</rt></ruby>子」について説明します。

## IV. <ruby>論理<rt>ろんり</rt></ruby><ruby>演算<rt>えんざん</rt></ruby>子（ろんりえんざんし）：複数の条件を<ruby>組<rt>く</rt></ruby>み<ruby>合<rt>あ</rt></ruby>わせる

「18歳以上、かつ、会員である」のように、複雑な判断を行いたいときに使うのが<ruby>論理<rt>ろんり</rt></ruby><ruby>演算<rt>えんざん</rt></ruby>子です。

### 1. 3つの基本ルール

1. <ruby>論理<rt>ろんり</rt></ruby><ruby>積<rt>せき</rt></ruby>（&&）: 「かつ（AND）」。すべての条件が true のときだけ全体が true になります。
2. <ruby>論理<rt>ろんり</rt></ruby>和（||）: 「または（OR）」。どれか1つでも true なら全体が true になります。
3. <ruby>論理<rt>ろんり</rt></ruby>否定（!）: 「ではない（NOT）」。結果を反対にします（true を false に変える）。

### 2. <ruby>真偽<rt>しんぎ</rt></ruby><ruby>値<rt>ち</rt></ruby>の結果（<ruby>真理<rt>しんり</rt></ruby><ruby>値<rt>ち</rt></ruby>表）

|条件A|条件B|A && B (かつ)|A \|\| B (または)|! A (ではない)|
|---|---|---|---|---|
|true|true|true|true|false|
|true|false|false|true|false|
|false|true|false|true|true|
|false|false|false|false|true|

複雑な条件をこれらの記号でシンプルに整理することが、ミスのない綺麗なコードを書くコツです。

## V. <ruby>演算<rt>えんざん</rt></ruby>の<ruby>優先<rt>ゆうせん</rt></ruby>度（えんざんのゆうせんど）：計算の正しい順番

1つの行にたくさんの<ruby>演算<rt>えんざん</rt></ruby>子があるとき、どの順番で処理されるかが重要です。順番を間違えると、計算結果がめちゃくちゃになってしまいます。

### 1. 基本のルール

算数と同じで、**「<ruby>掛<rt>か</rt></ruby>け<ruby>算<rt>ざん</rt></ruby>・<ruby>割<rt>わ</rt></ruby>り<ruby>算<rt>ざん</rt></ruby>（*, /）は、<ruby>足<rt>た</rt></ruby>し<ruby>算<rt>ざん</rt></ruby>・<ruby>引<rt>ひ</rt></ruby>き<ruby>算<rt>ざん</rt></ruby>（+, -）よりも先」** に計算されます。

### 2. 括弧 () でコントロールする

「<ruby>足<rt>た</rt></ruby>し<ruby>算<rt>ざん</rt></ruby>を先にしたい」というときは 括弧 () を使います。

* 10 + 2 * 5 → 20
* (10 + 2) * 5 → 60

### 3. プロとしてのコツ

1行で長く書くのではなく、括弧を使って順番をはっきりさせたり、計算をいくつかの行に分けたりしましょう。そうすることで、自分だけでなく、チームの仲間があなたのコードを読んだときに迷わなくて済むようになります。

## VI. まとめ：プログラミングの基礎を<ruby>固<rt>かた</rt></ruby>める

<ruby>演算<rt>えんざん</rt></ruby>子はJavaScriptで「<ruby>動<rt>うご</rt></ruby>き」を作るための大切な道具です。これらを使いこなせば、あなたのアイデアを自由にプログラムにすることができます。

**留学生が間違いやすいポイントをリマインド！**

* **= と === の<ruby>混同<rt>こんどう</rt></ruby>:** `=` は「<ruby>代入<rt>だいにゅう</rt></ruby>（右の値を左に入れる）」、`===` は「比較（同じか調べる）」です。
* **文字列の<ruby>結合<rt>けつごう</rt></ruby>:** 数字と文字を + でつなぐと文字になってしまいます。計算したいときは Number() を使いましょう。
* **<ruby>優先<rt>ゆうせん</rt></ruby>順位のミス:** 複雑な計算は括弧 () を使って順番を整理しましょう。

練習が一番の<ruby>近道<rt>ちかみち</rt></ruby>です

頭で覚えるだけでなく、実際に手を動かしましょう。ブラウザの「<ruby>開発<rt>かいはつ</rt></ruby>者ツール」にあるコンソールは、あなたの「最高の友達」です。 console.log() を使って、<ruby>演算<rt>えんざん</rt></ruby>の結果を自分の目で何度も確認してください。

* <ruby>例<rt>れい</rt></ruby>：console.log(10 * 5 === 50); と打って true が出るか試す。
