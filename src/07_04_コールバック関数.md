# コールバック<ruby>関数<rt>かんすう</rt></ruby>

<ruby>留学生<rt>りゅうがくせい</rt></ruby>のみなさんにもわかりやすいように、**「コールバック<ruby>関数<rt>かんすう</rt></ruby>」** をやさしい日本語で<ruby>説明<rt>せつめい</rt></ruby>します。

***

## I. コールバック<ruby>関数<rt>かんすう</rt></ruby>を一言（ひとこと）でいうと？

**「あとで やってね！という お<ruby>願<rt>ねが</rt></ruby>いのメモ」** です。

ふつうのプログラムは、上から下へ<ruby>順番<rt>じゅんばん</rt></ruby>に<ruby>進<rt>すす</rt></ruby>みます。
でも、コールバック<ruby>関数<rt>かんすう</rt></ruby>は「今すぐ」ではなく、**「ある<ruby>時<rt>とき</rt></ruby>が来たら」** 動くように<ruby>予約<rt>よやく</rt></ruby>する<ruby>命令<rt>めいれい</rt></ruby>です。

***

## II. 「お店でのちゅうもん」で たとえてみましょう

あなたがレストランへ行きました。

1. **あなた（メインプログラム）：** 店員さんに「オムライスをください。**できあがったら、ベルを鳴らしてください**」と言いました。
2. **店員さん（forEachメソッド）：** <ruby>料理<rt>りょうり</rt></ruby>を作ります。
3. **ベルを鳴らす（コールバック<ruby>関数<rt>かんすう</rt></ruby>）：** 料理ができたタイミングで、店員さんがベルを鳴らします。

この「できあがったら、〇〇してね」という **「〇〇」の部分** がコールバック<ruby>関数<rt>かんすう</rt></ruby>です。

***

## III. `forEach` の中での<ruby>動<rt>うご</rt></ruby>き

`forEach` は、<ruby>配列<rt>はいれつ</rt></ruby>（データのリスト）の中身をひとつずつ<ruby>見<rt>み</rt></ruby>ていく命令です。

```javascript
// 果物（くだもの）の名前のリスト
const fruits = ["りんご", "バナナ", "みかん"];

// fruits.forEach( ここに書くのがコールバック関数（かんすう） );
fruits.forEach((item) => {
  console.log(item + "を食べます");
});

```

### プログラムの中の会話（かいわ）

* **forEach:** 「リストの中身をひとつずつ取り出すよ！ 取り出すたびに、何をすればいい？」
* **コールバック<ruby>関数<rt>かんすう</rt></ruby>:** 「取り出したものに『を食べます』をつけて<ruby>表示<rt>ひょうじ</rt></ruby>してね！」
* **forEach:** 「わかった！ じゃあ、『りんご』を取り出したから、君（コールバック<ruby>関数<rt>かんすう</rt></ruby>）に<ruby>渡<rt>わた</rt></ruby>すね。<ruby>実行<rt>じっこう</rt></ruby>して！」
* **コールバック<ruby>関数<rt>かんすう</rt></ruby>:** 「はい！『りんごを食べます』と出します！」

これを、リストが終わるまで<ruby>繰り返<rt>くりかえ</rt></ruby>します。

***

## IV. なぜ「コールバック」という名前なの？

英語（えいご）の **"Call back"（電話をかけなおす）** と同じ意味です。

* **ふつうの<ruby>関数<rt>かんすう</rt></ruby>：** あなたが<ruby>関数<rt>かんすう</rt></ruby>を「呼ぶ（Call）」。
* **コールバック<ruby>関数<rt>かんすう</rt></ruby>：** 相手（forEachなど）が、あとであなたを「呼び出す（Call back）」。

このように、**「<ruby>主役<rt>しゅやく</rt></ruby>が<ruby>入<rt>い</rt></ruby>れ<ruby>替<rt>か</rt></ruby>わる」** のがポイントです。

***

### まとめ

* **コールバック<ruby>関数<rt>かんすう</rt></ruby>** ＝ 他のプログラムに「これ、あとで使ってね」と渡す<ruby>関数<rt>かんすう</rt></ruby>のこと。
* **forEach** ＝ リストのデータひとつずつに、コールバック<ruby>関数<rt>かんすう</rt></ruby>を動かしてあげる<ruby>役割<rt>やくわり</rt></ruby>。

## V. プログラム<ruby>例<rt>れい</rt></ruby>

「数字の<ruby>合計<rt>ごうけい</rt></ruby>」を<ruby>計算<rt>けいさん</rt></ruby>するコードを書いてみます。やさしい日本語で<ruby>説明<rt>せつめい</rt></ruby>しながら書きます。


### 1. <ruby>合計<rt>ごうけい</rt></ruby>を計算するプログラム

このプログラムは、リストの中にある数字を<ruby>全部<rt>ぜんぶ</rt></ruby><ruby>足<rt>た</rt></ruby>して、<ruby>最後<rt>さいご</rt></ruby>に答えを出します。

```javascript
// 1. 数字のリストを作ります
const numbers = [10, 20, 30, 40];

// 2. 合計を入れる「空（から）の箱」を作ります。最初は 0 です。
let total = 0;

// 3. forEach を使って、ひとつずつ足していきます
numbers.forEach((num) => {
  // ここがコールバック関数（かんすう）です！
  total = total + num; 
  console.log(num + " を足しました。今は " + total + " です。");
});

// 4. 最後に答えを表示（ひょうじ）します
console.log("ぜんぶの合計は " + total + " です！");

```

***

### 2. プログラムの<ruby>動<rt>うご</rt></ruby>きを<ruby>解説<rt>かいせつ</rt></ruby>

#### ① `let total = 0;`

これは、<ruby>計算<rt>けいさん</rt></ruby>の<ruby>結果<rt>けっか</rt></ruby>を入れておくための **「<ruby>貯金箱<rt>ちょきんばこ</rt></ruby>」** のようなものです。最初は何も入っていないので `0` にしておきます。

#### ② コールバック<ruby>関数<rt>かんすう</rt></ruby>の仕事

`numbers.forEach((num) => { ... });` の中（なか）が、お<ruby>願<rt>ねが</rt></ruby>いしたメモ（コールバック<ruby>関数<rt>かんすう</rt></ruby>）です。

* `forEach` がリストから `10` を見つける。
* コールバック<ruby>関数<rt>かんすう</rt></ruby>に `10` を<ruby>渡<rt>わた</rt></ruby>す。
* <ruby>関数<rt>かんすう</rt></ruby>が **「今の合計（0）に 10 を足してね！」** と<ruby>実行<rt>じっこう</rt></ruby>する。
* 次の `20` でも同じことをする。

これを<ruby>繰り返<rt>くりかえ</rt></ruby>すことで、貯金箱の中身がどんどん増えていきます。

***

### 3. もっと短く書く方法

プログラミングに<ruby>慣<rt>な</rt></ruby>れてきたら、こんな<ruby>書<rt>か</rt></ruby>き<ruby>方<rt>かた</rt></ruby>もできます。

* `total = total + num;`
* **`total += num;`** （これでも同じ意味になります！）
