# 三つのfor

## I. 古典的なfor

```JavaScript
for (let i = 0; i < 3; i++) {
  console.log(i + "番目：こんにちは！");
}
```

## II. <ruby>配列<rt>はいれつ</rt></ruby>の応用とforEach

大量のデータを<ruby>効率<rt>こうりつ</rt></ruby>よく処理するために、JavaScriptには forEach（フォーイーチ）という便利な道具があります。

### 1. forEachとコールバック<ruby>関数<rt>かんすう</rt></ruby>

以前、<ruby>関数<rt>かんすう</rt></ruby>は「<ruby>自動販売機<rt>じどうはんばいき</rt></ruby>」のようなものだと学びましたね。ボタンを押すと、中に入れた処理が実行されて結果が出てきます。

forEach を使うときは、この<ruby>関数<rt>かんすう</rt></ruby>をさらに便利に使います。

* <ruby>例<rt>たと</rt></ruby>え話: forEach は「店員さん」、渡す<ruby>関数<rt>かんすう</rt></ruby>は「店員さんに渡すチェックメモ（コールバック<ruby>関数<rt>かんすう</rt></ruby>）」です。 店員さんは、<ruby>配列<rt>はいれつ</rt></ruby>の中身（リンゴなど）をひとつ取るたびに、あなたが渡したメモを読んで「傷がないか確認する」などの作業を行ってくれます。

### 2. サンプルプログラム: <ruby>合計<rt>ごうけい</rt></ruby>を計算する

このプログラムは、リストの中にある数字を<ruby>全部<rt>ぜんぶ</rt></ruby><ruby>足<rt>た</rt></ruby>して、<ruby>最後<rt>さいご</rt></ruby>に答えを出します。

```javascript
// 1. 数字の「配列（はいれつ）」を作ります。
const numbers = [10, 20, 30, 40];

// 2. 合計を入れる「空（から）の箱」を作ります。最初は 0 です。
let total = 0;

// 3. forEach （上の例（たと）えでは「店員さん」）は、配列（はいれつ）の中身をひとつずつ取って、メモ通りに処理します。
numbers.forEach((num) => {
  // ここがコールバック関数（かんすう）です！ 店員さんに「この仕事をしてください」と渡すメモです！
  total = total + num; 
  console.log(num + " を足しました。今は " + total + " です。");
});

// 4. 最後に答えを表示（ひょうじ）します
console.log("ぜんぶの合計は " + total + " です！");

```

### 3. プログラムの<ruby>動<rt>うご</rt></ruby>きを<ruby>解説<rt>かいせつ</rt></ruby>

#### ① `let total = 0;`

これは、<ruby>計算<rt>けいさん</rt></ruby>の<ruby>結果<rt>けっか</rt></ruby>を入れておくための**「<ruby>貯金箱<rt>ちょきんばこ</rt></ruby>」**のようなものです。最初は何も入っていないので `0` にしておきます。

#### ② コールバック<ruby>関数<rt>かんすう</rt></ruby>の仕事

`numbers.forEach((num) => { ... });` の中（なか）が、お<ruby>願<rt>ねが</rt></ruby>いしたメモ（コールバック<ruby>関数<rt>かんすう</rt></ruby>）です。

* `forEach` がリストから `10` を見つける。
* コールバック<ruby>関数<rt>かんすう</rt></ruby>に `10` を<ruby>渡<rt>わた</rt></ruby>す。
* <ruby>関数<rt>かんすう</rt></ruby>が **「今の合計（0）に 10 を足してね！」** と<ruby>実行<rt>じっこう</rt></ruby>する。
* 次の `20` でも同じことをする。

これを<ruby>繰り返<rt>くりかえ</rt></ruby>すことで、貯金箱の中身がどんどん増えていきます。

So What?（なぜ大切なの？） 
| 従来の for 文 | forEach メソッド | 
| :--- | :--- | 
| <ruby>繰<rt>く</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>しの回数を自分で管理する | <ruby>配列<rt>はいれつ</rt></ruby>が自<ruby>動的<rt>どうてき</rt></ruby>に回数を決めてくれる | 
| コードが長くなりやすい | 短く、読みやすくなる |

コードが読みやすくなると、<ruby>開発<rt>かいはつ</rt></ruby>のスピードが上がり、ミスも減ります。

## III. <ruby>繰<rt>く</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>し処理の応用：for-in文

※ オブジェクトの性質を利用した応用的な<ruby>繰<rt>く</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>し処理です。

<ruby>配列<rt>はいれつ</rt></ruby>ではなく「オブジェクト」の中身（プロパティ）をすべて確認したいときは、for-in 文を使います。

|ループの種類|対象|使いどころ|
|---|---|---|
|for文|<ruby>数値<rt>すうち</rt></ruby>の回数、<ruby>配列<rt>はいれつ</rt></ruby>|決まった回数<ruby>繰<rt>く</rt></ruby>り<ruby>返<rt>かえ</rt></ruby>したいとき|
|while文|条件|条件が満たされるまで続けたいとき|
|for-in文|オブジェクトのプロパティ|オブジェクトの「キー」を全部調べたいとき|

So What?（なぜ大切なの？） オブジェクトの中にどんなデータがいくつ入っているか、事前にはわからない場合があります。for-in を使えば、どんなに項目が多くても漏れなくすべて処理できるため、<ruby>柔軟<rt>じゅうなん</rt></ruby>なプログラムが作れます。
